# Devlog — 2026-03-10: Table Consolidation (Phase 1)

## Summary

Consolidated ~15 field/form/option/domain tables into 5 unified tables, replacing
rigid per-domain models with a generic field-option-override architecture. This is
Phase 1 of the broader `dcplibrary/sfp` → `dcplibrary/requests` rename.

---

## Schema Changes

### New Tables

| Table | Purpose |
|---|---|
| `fields` | Unified field definitions (key, label, type, scope, condition, step, sort_order, etc.). Replaces `form_fields`, `custom_fields`, and domain lookups like `material_types`, `audiences`, `genres`. |
| `field_options` | Options belonging to a field (name, slug, metadata, sort_order, active, soft deletes). Replaces rows in `material_types`, `audiences`, `genres`, `custom_field_options`, etc. |
| `form_field_config` | Per-form field settings (required, visible, sort_order, etc.). Replaces `form_form_fields` and `form_custom_fields`. |
| `form_field_option_overrides` | Per-form option overrides (label_override, visible, sort_order). Unique on (form_id, field_id, option_slug). Replaces per-form option tables. |
| `request_field_values` | EAV storage for per-request field values (request_id, field_id, value text). Unique on (request_id, field_id). Replaces `material_type_id`, `audience_id`, `genre`, `where_heard` columns on `requests`, plus `request_custom_field_values`. |

### Removed Tables

`material_types`, `audiences`, `genres`, `consoles`, `form_fields`, `custom_fields`,
`custom_field_options`, `form_form_fields`, `form_custom_fields`,
`form_material_types`, `form_audiences`, `form_genres`,
`selector_group_material_type`, `selector_group_audience`,
`request_custom_field_values`

### Modified Tables

- `requests` — dropped `material_type_id`, `audience_id`, `genre`, `where_heard` columns (now EAV in `request_field_values`)
- `materials` — `material_type_id` → `material_type_option_id` (FK to `field_options`)
- `selector_groups` — pivots consolidated: `selector_group_material_type` + `selector_group_audience` → single `selector_group_field_option` pivot

## Model Changes

### New Models

- **Field** — unified field definition, scopes: `active()`, `ordered()`, `ofType()`, `forStep()`, `forScope()`
- **FieldOption** — belongs to Field, scopes: `active()`, `ordered()`
- **FormFieldConfig** — per-form field config, belongs to Form + Field
- **FormFieldOptionOverride** — per-form option override, belongs to Form + Field
- **RequestFieldValue** — EAV row, belongs to SfpRequest + Field

### Deleted Models (13)

MaterialType, Audience, Genre, Console, FormField, CustomField, CustomFieldOption,
FormFormField, FormCustomField, FormMaterialType, FormAudience, FormGenre,
RequestCustomFieldValue

### Updated Models

- **SelectorGroup** — `materialTypes()`/`audiences()` BelongsToMany → `fieldOptions()` via `selector_group_field_option`; added `fieldOptionsForKey()`
- **SfpRequest** — `scopeVisibleTo()` rewritten: uses `fields`/`field_options`/`request_field_values` joins with correlated subqueries against EAV
- **User** — `accessibleMaterialTypeIds()` + `accessibleAudienceIds()` → `accessibleFieldOptionIds(?string $fieldKey)`; table changed to `staff_users`
- **Form** — relationships updated to use FormFieldConfig/FormFieldOptionOverride
- **Material** — `material_type_id` → `material_type_option_id` (FK to `field_options`)
- **PatronStatusTemplate** — minor import cleanup

## Livewire Component Changes

### IllForm (public-facing)

- **Option helpers**: replaced 4 separate helpers (`getIllMaterialTypesOptions`, `getIllAudienceOptions`, `getIllGenreOptions`, `getIllCustomFieldOptions`) with generic `getFieldOptionsWithOverrides(int $fieldId, ?int $formId)` + `getOptionsForField(string $key)` + slim `getIllMaterialTypesOptions()` wrapper
- **Validation**: removed dead `formFieldRadioRule()`; unified `buildStepTwoRules()` to use `selectOrRadioRule()` for all field types
- **saveRequest()**: removed `audience_id`, `material_type_id`, `genre`, `where_heard` from `SfpRequest::create()`; Material::create uses `material_type_option_id`; replaced `RequestCustomFieldValue::insert()` with `RequestFieldValue::insert()` storing all step-two fields as EAV
- **render()**: unified `optionsByFieldId` collection for all select/radio fields; kept `audienceOptions`/`genreOptions` as view-compat aliases

### SfpForm + FormFields

- Updated to reference `Field`/`FieldOption`/`FormFieldConfig` instead of `FormField`/`CustomField`/`FormFormField`

### Admin Components (10 updated)

- **CustomFieldEdit** — Field replaces CustomField; removed MaterialType lookup
- **CustomFieldOptionsManager** — Field/FieldOption/RequestFieldValue replace old models
- **FormFieldEdit** — Field/FieldOption replace FormField/MaterialType/Audience/Genre/Console
- **FormFormFieldEdit** — FormFieldConfig/Field replace FormFormField/FormField
- **FormFormFieldOptionsManager** — FieldOption/FormFieldOptionOverride replace old option models
- **FormFormFieldOptionEdit** — FieldOption/FormFieldOptionOverride/FormFieldConfig replace old
- **FormCustomFieldEdit** — FormFieldConfig/Field replace FormCustomField/CustomField
- **FormCustomFieldOptionsManager** — FieldOption/FormFieldOptionOverride replace old
- **FormCustomFieldOptionEdit** — FieldOption/FormFieldOptionOverride replace old
- **OptionsManager** — Field replaces FormField in `buildLockedMap`
